The current main branch turns the initial preview into a much deeper read-side PostgreSQL workspace. These changes are available to source builds but do not have a tagged release yet.

## Operations

- Redesigned the instance overview around live metrics, connection capacity, server information, health checks, and replication context.
- Added a live Activity page and session inspector with query and transaction age, client origin, wait details, and blocking relationships.
- Added database Query Insights and improved extension inventory.

## Data explorer

- Added schema maps and detailed pages for table data, columns, keys, constraints, indexes, partitions, row-level security policies, triggers, and definitions.
- Added routines, sequences, types, collations, foreign-data wrappers and servers, replication objects, event triggers, and `pg_cron` discovery.
- Added SQL-style row filters, sorting, foreign-key navigation, per-table column layout persistence, and CSV, JSON, and SQL exports.
- Preserved PostgreSQL result types and null behavior while hardening streamed results and CSV output.

## Access and workflow

- Redesigned roles, memberships, grants, definitions, and effective-access maps.
- Added a command palette, keyboard shortcut registry, consistent filter toolbars, pagination, structured PostgreSQL errors, and stronger loading and partial-error states.
- Hardened onboarding lifecycle behavior, connection tests, credential handling, target policy, and embedded PostgreSQL startup.

See the [roadmap](/roadmap) for work that is planned but not on main.
