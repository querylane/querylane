## Navigate a schema

The schema overview summarizes tables, views, and other database objects. Use the map to understand foreign-key relationships, focus one table, and follow connected objects without losing the schema boundary.

The explorer also surfaces routines, sequences, types, collations, foreign-data wrappers and servers, replication objects, event triggers, and `pg_cron` jobs when they exist.

## Inspect a table

Table detail is split by the questions each catalog object answers:

| Tab | Use it for |
| --- | --- |
| Data | Read rows, filter and sort them, resize or persist columns, follow foreign keys, and export results |
| Columns | Check type, nullability, defaults, identity, generated values, and comments |
| Keys and constraints | Understand primary, unique, foreign, check, and exclusion rules |
| Indexes | Review index method, columns, predicates, expressions, uniqueness, and validity |
| Partitions | See partition parents, children, strategies, and bounds |
| Policies | Understand row-level security policies, roles, commands, and expressions |
| Triggers | Review trigger timing, events, enablement, functions, and definitions |
| Definition | Reproduce the object's PostgreSQL definition |

## Filter and export safely

Combine structured filters and sorts to narrow a page of rows. Querylane preserves PostgreSQL value types and nulls in its results. Export the current result as CSV, JSON, or SQL; CSV export neutralizes values that spreadsheet programs could interpret as formulas.

The current grid is for inspection, not editing. Editable cells and mutation workflows follow the safety-rail work described on the [roadmap](/roadmap).
